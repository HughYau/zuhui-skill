# 风格提取指南 / Style Extraction Guide

## 目的

从用户提供的过去汇报样本中提取个人写作风格特征，生成风格摘要（style summary）保存到配置文件，用于后续生成时模仿。

## 分析维度

读取用户提供的1-3份样本文件，从以下维度提取风格特征：

### 1. 句式偏好
- 平均句子长度（短句为主 / 长句为主 / 混合）
- 是否使用bullet points / 编号列表
- 是否倾向用完整句子还是短语式表达
- 是否有嵌套结构（括号补充说明等）

### 2. 用词习惯
- 正式程度（学术正式 / 口语随意 / 混合）
- 领域术语使用频率
- 是否中英混合（如中文中夹英文术语）
- 常用的连接词/过渡词
- 有没有口头禅式的表达

### 3. 结构偏好
- 先说结果还是先说过程
- 是按时间线组织还是按主题组织
- blockers/problems 放在哪里（夹在中间 / 单独一节）
- next steps 的详略程度

### 4. 开头/结尾习惯
- 称呼方式（"老师好" / "Hi Prof. X" / 直接开始 / "Dear..."）
- 开头是直奔主题还是有寒暄
- 结尾签名格式
- 结束语用什么（"祝好" / "Best" / "Thanks" / 无）

### 5. 标点和格式习惯
- 标点使用习惯（句号多还是逗号多、用不用感叹号、省略号等）
- 是否用emoji（以及频率和类型）
- 缩进和换行风格
- 英文中是否用缩写（don't / doesn't vs do not）

## 风格摘要格式

提取完成后，生成如下格式的摘要，写入 `.progress-config.yaml` 的 `style.style_summary`：

```yaml
style_summary:
  sentence_style: "短句为主，偶尔用括号补充，不太用bullet points"
  formality: "偏口语，中文夹英文术语"
  structure: "先说结果，再说过程，problems放最后"
  opening: "老师好，直接说事"
  closing: "祝好 + 名字"
  punctuation: "句号少，逗号多，偶尔用波浪号"
  notable_habits: "喜欢用'跑了'代替'运行了'，数字不加单位直接说"
```

## 增量学习

用户后续提供更多样本时：
1. 读取现有 style_summary
2. 分析新样本
3. 如果发现新的风格特征，更新对应维度
4. 如果新样本和旧摘要有冲突，以更多样本支持的风格为准
5. 更新后保存

## 使用时机

- **生成内容时**：先读style_summary获取快速指导，再参考原始样本中的具体表达
- **去AI化时**：如果style_summary显示用户偏口语，去AI化时就往口语方向调整
- **无样本时**：跳过风格适配，仅做去AI化处理
